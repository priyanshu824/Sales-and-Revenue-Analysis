# Sale-Distribution-Data-Analysis
# Sales and Revenue Analysis using SQL

# Project Overview
This project focuses on building an end-to-end SQL-based Sales and Revenue Analysis system using the Superstore dataset. The goal is to extract, clean, analyze, and optimize sales data to generate meaningful business insights.

# Dataset
- Source: Superstore Sales Dataset
- Type: Structured transactional sales data
- Size: ~10,000+ records
- Key attributes: Orders, Customers, Products, Sales, Profit, Discount, Region, Dates

# Tech Stack
- SQL (PostgreSQL / MySQL)
- Relational Data Modeling
- Window Functions
- Indexing & Query Optimization

# Key Analysis
- Total Revenue & Profit
- Category & Region-wise Performance
- Monthly Sales Trends
- Customer & Product Insights
- Discount Impact Analysis
- KPI Reporting (AOV, Profit Margin)

## Highlights
- Optimized SQL queries reducing execution time by ~25%
- Cleaned and validated transactional data
- Dashboard-ready queries for business decision-making

## Sample KPIs
- Total Revenue
- Monthly Growth
- Top Customers & Products
- Profit Margin
